🟡 Parte II – Questões Discursivas (11–15)

11. Explique o conceito de Programa Armazenado e sua importância histórica.
Resposta: O conceito de Programa Armazenado determina que as instruções que o computador deve executar (o software) sejam armazenadas eletronicamente na mesma memória de leitura/escrita que os dados. Historicamente, isso revolucionou a computação, pois eliminou a necessidade de reconfigurar cabos ou painéis físicos para mudar a tarefa da máquina. O hardware tornou-se universal, e a funcionalidade passou a ser definida puramente pelo software.
12. Descreva os quatro componentes principais da Arquitetura de Von Neumann.
Resposta: 1) Unidade Central de Processamento (CPU): composta pela ULA e Unidade de Controle, é o cérebro que processa os dados.
2) Memória Principal: armazena temporariamente os dados e as instruções do programa em execução.
3) Dispositivos de Entrada: permitem a inserção de dados externos no sistema (ex: teclado, mouse).
4) Dispositivos de Saída: recebem os dados processados e os apresentam ao usuário (ex: monitor, impressora).
13. Diferencie a função da ULA e da Unidade de Controle.
Resposta: A Unidade Lógica e Aritmética (ULA) é a "calculadora" do sistema, realizando o trabalho bruto: somas, subtrações e operações lógicas booleanas (AND, OR, NOT). Já a Unidade de Controle (UC) atua como o "maestro" do processador: ela busca as instruções na memória, decodifica o que deve ser feito e envia sinais elétricos instruindo a ULA e a memória sobre como agir para completar aquela instrução.
14. Explique com suas palavras o que é o Gargalo de Von Neumann.
Resposta: É a limitação de desempenho que ocorre porque a CPU trabalha em velocidades altíssimas, mas a memória RAM é relativamente lenta e está separada fisicamente. Como ambos compartilham o mesmo barramento (via de comunicação) para buscar tanto instruções quanto dados, a CPU passa grande parte do seu tempo "ociosa", apenas esperando a informação trafegar da memória até ela para poder trabalhar.
15. Por que a arquitetura original pode representar risco à segurança da informação?
Resposta: Porque a arquitetura não distingue fisicamente o que é um "dado" do que é uma "instrução" — ambos são apenas sequências de bits na mesma memória principal. Isso significa que, se um atacante conseguir inserir uma sequência de bits maliciosos em uma área destinada a dados de usuário, ele pode manipular a CPU para ler e executar essa sequência como se fosse um código legítimo do programa.

🔵 Parte III – Questões Aplicadas e Reflexivas (16–20)

16. Explique como o conceito de cache melhora o desempenho de servidores Back-End.
Resposta: O conceito de cache no hardware (armazenar dados frequentemente acessados perto da CPU para evitar o gargalo da RAM) é aplicado em software no Back-End. Ferramentas como Redis ou Memcached armazenam em memória RAM ultrarrápida os resultados de consultas frequentes ao Banco de Dados (que fica no disco rígido e exige processamento pesado). Assim, a aplicação Back-End evita a "latência de busca" no disco, melhorando o tempo de resposta da API de forma análoga ao que a memória cache L1/L2 faz pela CPU.
17. Relacione a Arquitetura de Von Neumann com a execução de um programa em Java.
Resposta: Quando um código Java é executado, a Máquina Virtual Java (JVM) carrega tanto o programa (o bytecode compilado) quanto as variáveis do usuário para a Memória RAM. O processador, atuando sob os preceitos de Von Neumann, busca sequencialmente essas instruções do bytecode e os dados na RAM, processa na ULA e devolve o resultado para a mesma memória unificada.
18. Descreva como um ataque de injeção de código pode explorar a arquitetura de programa armazenado.
Resposta: No ataque de Buffer Overflow, um software mal programado permite que dados externos (como um texto digitado em um formulário) ultrapassem o tamanho reservado na memória. Como dados e instruções dividem o mesmo espaço linear (característica de Von Neumann), o excesso de dados transborda e sobrescreve o endereço de retorno da função. Quando a CPU vai buscar a próxima instrução, ela é redirecionada para a área de memória do dado transbordado, executando o código injetado pelo hacker.
19. Compare, de forma resumida, a Arquitetura de Von Neumann com a Arquitetura Harvard.
Resposta: A Arquitetura de Von Neumann utiliza uma única memória e um único barramento compartilhado para instruções e dados, o que é altamente flexível e otimiza o espaço, mas gera um gargalo de tráfego. A Arquitetura Harvard utiliza memórias físicas independentes e barramentos duplos separados (um para dados, outro para código), eliminando o gargalo, pois permite o acesso simultâneo a ambos, embora torne a construção do circuito mais complexa e cara.
20. Analise como os princípios de Confidencialidade e Integridade podem ser afetados por vulnerabilidades estruturais da arquitetura.
Resposta: Como a memória é um grande bloco linear unificado, o hardware base não sabe a quem pertencem os bits ali armazenados. Se não houvesse proteções de software (como a Unidade de Gerenciamento de Memória - MMU do Sistema Operacional), um programa poderia ler os dados de outro na mesma RAM (quebrando a Confidencialidade) ou alterar intencionalmente os dados ou o código de um programa concorrente (quebrando a Integridade). Soluções como Anéis de Proteção e bits de não-execução (NX Bit) foram criadas justamente para mitigar essas falhas estruturais.
